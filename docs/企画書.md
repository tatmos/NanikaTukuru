# NanikaTukuru 音響ラボ — 企画書

## 目的

[note記事「元エンジニアがAIでバイブコーディングしてみた話」](https://note.com/tatmos/n/n92960b7b18da) のサンプルとして、

- 作業フォルダに企画・コード・ドキュメントを一箇所にまとめる
- Cursor Agent とのやり取りを残しながら進める
- **GitHub Pages（github.io）でそのまま公開できる**静的サイトにする
- 同じ機能を **Godot 4（Windows）** でも動かす（併存）
- 同じ機能を **JUCE（VST3 / Standalone）** でも動かす（併存）

を満たすミニ音響ラボを作る。

## コンセプト

ブラウザだけで動く「机の上の小さな信号デスク」。  
外部サーバーやビルド手順なしで、オシレータ／ノイズ／フィルタ／ディレイを触りながら、波形とスペクトルを見て遊べる。

## 機能スコープ（v1）

1. オシレータ（sine / square / sawtooth / triangle）+ 周波数・音量
2. ホワイトノイズ（量を混ぜられる）
3. ローパス／ハイパス風の簡易フィルタ
4. ディレイ（時間・フィードバック）
5. リアルタイム波形・スペクトル表示
6. 再生／停止（AudioContext はユーザー操作後に開始）

対象外（あえてやらない）:

- マイク入力・録音の保存
- アセット生成AI連携
- npm ビルドやフレームワーク

## 技術方針

### Web（v1）

- 純粋な静的ファイル（HTML / CSS / JS）
- Web Audio API + Canvas 2D
- GitHub Pages: リポジトリルートを公開（`https://tatmos.github.io/NanikaTukuru/`）
- 相対パスのみ（サブパス公開でも壊れないようにする）

### Godot（v1 移植）

- Godot 4.2+ / Desktop（Windows 想定）
- `AudioStreamGenerator` でオシレータ＋ノイズ
- オーディオバス `Lab` に Filter / Delay / SpectrumAnalyzer
- UI は Control ノードをスクリプトで構築（Web 版と同じ操作項目）
- Web 版は残し、`godot/` 配下に併存

### JUCE（v1 移植）

- JUCE 8 + CMake（FetchContent で JUCE 取得）
- フォーマット: **VST3** + **Standalone**
- シンセ（自前で波形生成）として実装
- `StateVariableTPTFilter` + `DelayLine` + APVTS
- Web / Godot は残し、`juce/` 配下に併存

## 進め方（記事デモ用）

1. 企画をドキュメントに書く
2. Agent に実装を依頼し、やり取りを `docs/agent-log.md` に残す
3. ローカルで Web / Godot / JUCE それぞれ確認
4. push 後、GitHub Pages を有効化して Web を公開
5. Godot は Export で Windows `.exe`（任意）
6. JUCE は CMake で VST3 / Standalone をビルド（任意）
